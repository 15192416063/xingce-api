# 验收测试清单

> 用途：技术人员开发完，对照本清单自测；你（产品方）验收时也以此为准。
> 每项标注「怎么测」和「通过标准」，避免「做完了但不是想要的」。

## 一、切题功能

- [ ] **能正确切出题目**
  - 怎么测：上传一份文字版行测 PDF，看切出的题数是否合理。
  - 通过标准：题号连续、每道题的题干和选项完整，无明显漏切或多切。

- [ ] **图形题被正确过滤**
  - 怎么测：用含图形推理题的卷子测试。
  - 通过标准：图形推理题（"填入问号处…"那种）被自动跳过，但类比推理题（如"司机：驾驶"这种短题）正常保留，没被误删。

- [ ] **申论/材料题被截断**
  - 通过标准：卷子末尾的申论、材料处理题没有被当成普通题切进来。

## 二、考点提炼

- [ ] **提炼出的考点合理**
  - 怎么测：抽查几道题，看 LLM 提炼的考点摘要。
  - 通过标准：格式是「题型+考点+主题」，且与题目实际考点相符，不是简单复述题干。

## 三、相似题检索（核心功能）

- [ ] **能找到考点相似的题**
  - 怎么测：先入库一批题，然后输入一道库里没有但考点相同的题，看返回结果。
  - 通过标准：返回的题考点确实相近，不是仅仅字面词重合。
  - 进阶验证：输入一道"基层治理"主题的题，应能匹配到其他"基层治理"题，即使它们讲的具体事不同。

- [ ] **无相似题时正确提示**
  - 怎么测：输入一道和库里完全不相关的内容。
  - 通过标准：返回"未找到足够相似的题"，而不是硬凑几道不相关的。

- [ ] **相似度阈值生效**
  - 通过标准：调高/调低阈值，返回结果数量相应变化。

## 四、双库与隔离（产品化阶段重点验）

- [ ] **公共库全员可搜**
  - 通过标准：管理员上传到公共库的题，任何用户都能搜到。

- [ ] **私有库隔离生效（隐私红线）**
  - 怎么测：用户 A 上传私有题，用户 B 登录后检索。
  - 通过标准：B 绝对搜不到 A 的私有题。这一条不通过 = 严重缺陷，不可上线。

- [ ] **管理员权限受控**
  - 怎么测：普通用户尝试上传到公共库。
  - 通过标准：被拒绝。只有管理员能写公共库，且校验在后端（不能只靠前端隐藏按钮）。

## 五、模型与配置

- [ ] **建库和检索用同一 embedding 模型**
  - 通过标准：确认配置一致。若不一致，检索结果会完全错乱。

- [ ] **API key 不在代码里硬编码**
  - 通过标准：key 通过 .env / 环境变量读取，代码仓库里看不到真实 key。

## 六、容错

- [ ] 上传非 PDF / 损坏文件 → 有友好报错，不崩溃。
- [ ] 上传扫描版 PDF（提取不出文字）→ 有提示，不静默失败。
- [ ] API 调用失败（超时/欠费）→ 有错误提示，不卡死。

---

## 验收总结判断

**MVP 阶段最关键的三条**（这三条过了，核心价值就成立）：
1. 切题切得对（第一、二项）
2. 相似题找得准、且字面不同也能匹配（第三项核心验证）
3. 无结果时如实提示，不硬凑（第三项）

**产品化前必须额外过**：私有库隔离（第四项），这是隐私和合规红线。
