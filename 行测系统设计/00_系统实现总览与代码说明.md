# 行测智能题库 · 系统实现总览与代码说明（当前真实架构）

> 本文是系统的**权威实现文档**，描述实际跑起来的系统，以代码为准，逐文件说明「干什么、怎么实现」。
> （早期那套 Java/Spring + MySQL + 会员定价的设计草案已废弃删除——实际走的是更轻的 Python/FastAPI + SQLite/Chroma。）

---

## 一、一句话定位与技术栈

一个**免费的行测刷题平台**：管理员上传真题 PDF → 系统自动切题、抠图、分类、建向量入库 →
用户登录后按题型刷题、整卷模考、AI 辅导对话调题、上传自己的题/答案。

| 层 | 选型 | 说明 |
|---|---|---|
| 前端 | **原生 HTML/CSS/JS**（单文件 `static/index.html`） | 无框架、无构建，后端当静态文件托管 |
| 后端 | **FastAPI + Uvicorn**（ASGI） | 路由全在 `main.py`；systemd 跑 `uvicorn main:app --port 8000` |
| 关系库 | **SQLite + SQLAlchemy**（WAL） | `xingce_demo.db`；`XC_DB_URL` 可切 MySQL |
| 向量库 | **Chroma**（PersistentClient，本地持久化） | 存「考点摘要」向量，按 `scope/l2` 过滤检索 |
| LLM | **DeepSeek**（`deepseek-chat`，OpenAI 兼容） | 多渠道自动故障切换（`ai_channel` 表） |
| Embedding | **bge-large-zh-v1.5**（硅基流动 API，1024 维） | 走 API 不本地跑模型，省内存便于上云 |
| PDF/抠图 | **PyMuPDF(fitz)** + `pymupdf4llm` + NumPy/Pillow | 切题转 Markdown；抠图后处理漂白/裁切 |
| 导出 PDF | **reportlab** | 题目集 + 答案页，中文字体自动探测 |

依赖原则：**能用标准库就不引第三方**（认证、限流、备份全是标准库手写），部署简单。

## 二、整体架构与三条核心数据流

```
浏览器(index.html)
   │  fetch /api/*
   ▼
FastAPI(main.py) ── 中间件:三层限流 + 安全头 + HTML不缓存
   ├─ auth.py        令牌校验/权限(current_user / require_admin)
   ├─ security.py    限流/爆破锁定/每日备份
   ├─ stats.py       在线心跳/PV/UV/token分账
   │
   ├─ ingest.py ──── 入库编排(后台任务)
   │     ├─ ai.split_questions   文字题切题(LLM)
   │     ├─ extract_graphics     图形题抠图(本地)
   │     ├─ extract_material     资料分析材料组(本地)
   │     ├─ ai.classify_batch    分类打标(LLM)
   │     └─ vectors.upsert       建向量(embedding API → Chroma)
   │
   ├─ ai.py          切题/分类/解题/对话/OCR/embedding(多渠道)
   ├─ vectors.py     Chroma 读写(scope+题型过滤)
   ├─ pdfgen.py      题目集导出 PDF
   └─ db.py          所有表模型 + 自动迁移
        │
   SQLite(xingce_demo.db) + Chroma(vector_db/) + 抠图(data/images/)
```

### 流程① PDF 入库（管理员上传 → 后台异步）
1. `POST /api/ingest/upload`：存 PDF，建 `IngestionJob`（状态机 0待处理→1解析→2入库→3完成/4失败），**立即返回 job_id**，前端轮询 `/api/ingest/job/{id}` 看进度。
2. 后台 `ingest.run(job_id)`：
   - 建 `Paper`（一份 PDF = 一套卷）；扫全卷算「跨页重复噪声」（页眉页脚水印）。
   - **资料分析**：`extract_material.parse` 本地解析材料组（共享材料+图表，挂多道小题）。
   - **图形题**：`extract_graphics.extract` 本地抠图（排除资料分析小题）。
   - **文字题**：`ai.split_questions`（LLM 按题号边界分批切题）→ `ai.classify_batch`（10题/次分类）。
   - **缺号校验闭环**：全文题号锚点 vs 实际切出的题号，缺的定点重切一次。
   - 去重（指纹）→ 落库 → `vectors.upsert` 建向量。
   - 公共库的图形/资料分析题入 `status=0 待确认`（管理员核对裁剪图后再进池）；用户私库直接入池。

### 流程② 答案上传（一卷的答案单独传）
`POST /api/papers/{pid}/answers`：粘贴答案文本或传答案 PDF → `ai.parse_answer_key`
（先本地正则解析「1-5 ABCDA」「1.A」等紧凑排版**免 token**，带解析的长文档才交 LLM）→ 按题号回填 `answer/explanation`，标 `answer_origin=official`。

### 流程③ 对话调题 / AI 解题（用户侧，要快）
- `POST /api/ai/ask`：`ai.chat` 出辅导话术；同时 `ai.quick_classify`（词表秒级、免费）识别题型意图，
  `_mine_first_recommend` **先私库按题型直查 → 向量检索补足 → 公共库按题型兜底**，把真题卡片返回。
  **题目调用走向量检索/直查，不临时让 LLM 编题**——入库时已分好类、建好向量，调用即命中。
- `POST /api/questions/{qid}/ai-answer`：无答案的题让 `ai.solve` 生成答案+解析，**写库标 `origin=ai`、全员共享**，下次直接读缓存不重复烧钱。

## 三、逐文件说明

### 后端核心（`xingce_api/`）

| 文件 | 作用 | 关键实现 |
|---|---|---|
| **main.py** | FastAPI 应用 + 全部 HTTP 路由（~70 个） | 启动即 `init_db()`、起备份线程、`_startup_recover`（DB 配置覆盖 + 中断任务标失败）；HTTP 中间件做**三层限流**（认证/AI/全局）+ 安全响应头 + HTML 不缓存；`/img` 静态挂载抠图；全局异常兜底不泄露堆栈；管理面板可热改的配置走 `RUNTIME_SETTINGS` 白名单 |
| **config.py** | 集中配置 | 读环境变量 → 退回同目录 `.env`（手写极简解析，免装 dotenv）；定义 DB/IMAGE/VECTOR 路径、AI/embedding key、TTL、限流阈值、每日 AI 上限、价格等 |
| **db.py** | 数据模型 + 会话 + 迁移 | SQLAlchemy 全部表（见第四节）；SQLite 开 WAL（并发+抗崩）；`_migrate` 轻量自动补列（旧库加新字段不报错）；`_backfill_papers` 老库回填套卷；`init_db` 建表 |
| **auth.py** | 认证 | 密码 **PBKDF2-HMAC-SHA256 + 每用户盐**；令牌自签（HMAC-SHA256，JWT 风格，带 `exp`+`ver`）；`token_ver` 版本号——登录/改密+1，**旧令牌(含被盗的)立即全失效**；FastAPI 依赖 `current_user/require_admin/optional_user` |
| **security.py** | 安全防护 | 滑动窗口限流（内存 deque）；登录爆破锁定（N 次错锁 M 秒）；真实 IP（`X-Forwarded-For`）；**SQLite 每日在线备份 + `PRAGMA quick_check` 完整性自检**（损坏在面板亮红灯），保留最近 N 天 |
| **ai.py** | 所有 AI 能力 | **多渠道故障切换**（`ai_channel` 按优先级，坏了换下一个，没配渠道退回 `.env`）+ **token 分账**（按用途场景+渠道记账）；`split_questions`（题号边界分批切题）、`classify`/`classify_batch`（结构化分类+细分考点）、`quick_classify`（词表零成本意图识别）、`solve`（解题）、`chat`（辅导对话，**禁止 AI 自己编题**）、`parse_answer_key`（先本地正则后 LLM）、`ocr_page`（视觉渠道扫描件 OCR）、`embed`（embedding API，带 500 条查询缓存） |
| **vectors.py** | 向量库封装 | Chroma `upsert/delete/search`；`search` 支持 **scope 过滤 + l2 题型过滤**（避免图形推理串到资料分析） |
| **stats.py** | 运营统计 | 在线心跳（内存，5 分钟 TTL）；`StatDaily`（PV/对话/token 总量）、`VisitDay`（UV）、`TokenStat`（按 日/用途/渠道 分账）；**所有写入吞异常，统计绝不拖垮业务** |
| **ingest.py** | 入库编排 | 见流程①；状态机 + 进度回写 + 异常标失败 + 空卷标删；扫描件无文字层时走 `ai.ocr_page` |
| **textutil.py** | 文本清洗 | **自适应跨页噪声检测**（过半页面重复的短行 = 页眉页脚水印，不靠关键词）+ 正则噪声库（页码/机构/网址/「祝您上岸」）；入库与 API 返回都调用（幂等） |
| **pdfgen.py** | 导出 PDF | reportlab 生成「题目集 + 答案页」；中文字体按候选路径自动探测（系统字体→内置 CID 兜底）；资料分析共享材料同组只打印一次 |
| **reembed.py** | 重建向量 | 换 embedding 模型/维度后：删旧 collection + 重算全部题向量 |
| **exams.py** | 全国考试日历 | 内置各省 seed 数据（历年规律兜底）+ 管理员一键抓取（公开页面 + AI 提取）；写 `ExamInfo` |

### 抠图引擎（项目根目录，被 `ingest.py` 复用）

| 文件 | 作用 | 关键实现（**本轮重点迭代**） |
|---|---|---|
| **extract_graphics.py** | 图形题抠图 | 题号 y 锚定 → 区域内「真图形对象」自检（位图/矢量，按面积/高度/全宽剔除文字行）→ **遇资料分析/材料分界行截断带**（不吞下游饼图）→ 裁剪上沿压到首个图形顶部（**题干不进图**）→ 题干/文字型选项/成句散文/页眉页脚**整行涂白** → 浅中灰水印漂白 → **autocrop 贴紧内容包围盒** |
| **extract_material.py** | 资料分析材料组 | 事件法分组（小题+图表簇按阅读顺序）→ 图表聚簇（间距过大**或两图间有题号/选项/材料组标号边界即断开**，防串题）→ 裁剪图表（同样涂白题干/选项/散文、漂白水印、autocrop） |

### 运维/部署脚本

| 文件 | 位置 | 作用 |
|---|---|---|
| **batch_ingest_public.py** | xingce_api/ | 批量把「行测-真题」灌公共库（复用 `ingest.run`），幂等、`--since 年份` 过滤、`--copy` 可选复制 PDF |

| **deploy.sh** | xingce_api/ | 服务器**代码**更新：`git pull`→装依赖→重启 |
| **更新代码.bat** | 根目录 | 本地一键 `git add -A` + commit + push |
| **导出数据.bat** | 根目录 | 本地一键跑 `export_public.py` 出种子 zip |

## 四、数据模型（核心表）

| 表 | 作用 | 关键字段 |
|---|---|---|
| **Paper** | 套卷（一份 PDF=一套卷） | scope(public/user:N)、title、question_count、answer_count、status |
| **Question** | 题目核心 | paper_id、material_id、scope、category_l1/l2、knowledge_point、difficulty、content、**answer**、**answer_origin**(official/ai)、explanation、has_image、**vector_id**、**fingerprint**(去重)、confidence、status(1正常/0待确认/2删) |
| **QuestionImage** | 题目关联图 | question_id、object_key(相对 IMAGE_DIR) |
| **MaterialGroup** | 资料分析材料组 | scope、material_text、image_keys(竖线分隔多图) |
| **IngestionJob** | 入库任务（状态机） | status/progress/total/done/dup/graphic/missing_nums/error_msg |
| **User / InviteCode** | 账号 / 邀请码 | password_hash、role、status、**token_ver**(吊销) |
| **PracticeRecord / WrongBook / Favorite / MockExam** | 做题记录 / 错题本(间隔重复) / 收藏 / 整卷模考 | — |
| **AiChannel / Setting** | AI 渠道 / 运行时配置 | api_key、priority、supports_vision |
| **StatDaily / VisitDay / TokenStat** | 运营统计 / UV / token 分账 | day、scene、channel、calls、tokens_in/out |
| **AuditLog / LoginLog / AiUsage / ExamInfo / News** | 审计 / 登录日志 / 每日 AI 计数 / 考试日历 / 资讯 | — |

关系：`Paper 1—N Question`；`MaterialGroup 1—N Question`（资料分析）；`Question 1—N QuestionImage`；
`Question.vector_id == str(question_id) == Chroma 里的 id`（三者一致，重映射时要同步）。

`scope` 约定：`public`=公共题库（全员可用），`user:{id}`=某用户私有题库。

## 五、关键设计点（为什么这么做）

- **省钱三段式**：词表秒分类（免费）→ 贴题才调 LLM → 入库分类 10 题/次批处理；AI 解题/答案结果**入库缓存全员共享**，同题不重复花钱。
- **调题快**：入库时就分类+建向量，调用走向量检索/按题型直查，**不临时算**。
- **抠图不靠 VLM 猜坐标**：坐标全来自 PDF 文本流（题号 y）+ 图形对象自检，稳定可复现；后处理统一漂白水印、涂白题干/选项/散文、autocrop 贴紧。
- **可靠性**：任务状态机 + 启动自愈（中断任务标失败）+ 每日备份 + 完整性自检 + 错误日志轮转 + 统计写入吞异常。
- **安全**：密码哈希、令牌签名+版本吊销、三层限流、爆破锁定、安全响应头/CSP、管理操作审计；**种子包导出剔除全部隐私表**。

## 六、部署与数据搬运

- **代码**走 git：本地 `更新代码.bat` → 服务器 `bash deploy.sh`（= `/root/update.sh`）。
- **数据**走 GitHub Release（不进 git，含密码哈希且体积大）：本地 `导出数据.bat` 出 `public_seed_*.zip` → 传 Release → 服务器 `bash update_data.sh`（下载→解压→`import_public.py --replace`→`reembed.py`→重启）。
- **服务器目录** `/root/xingce/xingce_api`，venv 在 `venv/`，服务名 `xingce`（systemd），端口 8000。
- **必配 `.env`**：`DEEPSEEK_API_KEY`（LLM）、`XC_EMBED_KEY`（embedding，建向量必需）、`XC_SECRET_KEY`（令牌签名，上线务必设随机长串）、`XC_DB_URL`（默认 SQLite）。

## 七、待改进 / 已知边界

- `extract_material._find_section` 对**老版式(2003 等)**「根据所给图形回答问题」识别不到，老卷资料分析图可能漏抓（公共库已只保留近 10 年，影响小）。
- 限流/在线统计为**单进程内存实现**，多 worker 部署需按比例放宽阈值或换 Redis。
- SQLite 适合当前规模；用户量大时按 `XC_DB_URL` 切 MySQL。
