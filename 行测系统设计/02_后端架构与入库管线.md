# 后端架构与入库管线骨架(Spring Boot 3)

> 目标:用户传 PDF → 立即返回 jobId → 后台异步切题/分类/算向量/存图入库 → 前端轮询进度。
> 关键词:**任务落库 + 队列 + 线程池 worker + 幂等去重 + 失败重试**。

## 一、分层结构

```
com.xingce
├── config        线程池、Redis、MinIO、WebClient 配置
├── controller    UploadController / PracticeController / JobController
├── service
│   ├── IngestionService   入库编排(状态机)
│   ├── PdfService         PDFBox 提文字 + 渲染页成图
│   ├── AiClient           调 DeepSeek(切题/分类) + 通义(embedding/qwen-vl)
│   ├── VectorService      Redis Stack 向量写入/检索
│   ├── StorageService     MinIO 上传/取对象
│   ├── QuotaService       会员额度原子扣减(Redis)
│   └── QuestionService    出题/相似检索/判分
├── entity + mapper        MyBatis-Plus
└── mq                     Redis Stream 生产/消费
```

## 二、上传入口:立刻返回,不阻塞

```java
@RestController
@RequestMapping("/api/ingest")
@RequiredArgsConstructor
public class UploadController {
    private final StorageService storage;
    private final IngestionJobMapper jobMapper;
    private final StringRedisTemplate redis;
    private final QuotaService quota;

    @PostMapping("/upload")
    public R<Long> upload(@RequestParam MultipartFile file, @AuthUser Long userId) {
        if (!file.getOriginalFilename().toLowerCase().endsWith(".pdf"))
            return R.fail("仅支持 PDF");
        if (!quota.hasUploadQuota(userId))         // 额度预检(真实扣减在入库时按实际题数)
            return R.fail("上传额度不足,请升级会员");

        // 1) 原始 PDF 存 MinIO
        String objectKey = storage.put("pdf/" + userId + "/" + UUID.randomUUID() + ".pdf", file);

        // 2) 建任务(状态=待处理)——这是可靠性的"账本"
        IngestionJob job = new IngestionJob();
        job.setUserId(userId);
        job.setFileName(file.getOriginalFilename());
        job.setFileObjectKey(objectKey);
        job.setStatus(0);
        jobMapper.insert(job);

        // 3) 投递到队列(Redis Stream),worker 异步消费
        redis.opsForStream().add("ingest:stream",
                Map.of("jobId", job.getId().toString()));

        return R.ok(job.getId());   // 立即返回 jobId,前端拿去轮询进度
    }
}
```

## 三、线程池配置(worker 的执行层)

```java
@Configuration
@EnableAsync
public class ThreadPoolConfig {
    @Bean("ingestExecutor")
    public ThreadPoolTaskExecutor ingestExecutor() {
        ThreadPoolTaskExecutor ex = new ThreadPoolTaskExecutor();
        ex.setCorePoolSize(4);              // 入库是 IO/网络密集(等LLM),核数可略大
        ex.setMaxPoolSize(8);
        ex.setQueueCapacity(100);
        ex.setThreadNamePrefix("ingest-");
        ex.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());
        ex.initialize();
        return ex;
    }
}
```

## 四、队列消费者:把任务交给线程池,失败可重试

```java
@Component
@RequiredArgsConstructor
public class IngestStreamConsumer {       // 启动时注册 StreamMessageListenerContainer
    private final IngestionService ingestion;
    private final IngestionJobMapper jobMapper;

    public void onMessage(MapRecord<String,String,String> msg) {
        Long jobId = Long.valueOf(msg.getValue().get("jobId"));
        try {
            ingestion.process(jobId);                 // 真正干活(@Async ingestExecutor 内)
        } catch (Exception e) {
            IngestionJob job = jobMapper.selectById(jobId);
            if (job.getRetryCount() < 3) {            // 有限重试
                job.setRetryCount(job.getRetryCount() + 1);
                jobMapper.updateById(job);
                // 重新入队(可加退避延迟)
            } else {
                markFailed(jobId, e.getMessage());
            }
        }
        // ACK
    }
}
```

## 五、核心:入库编排(状态机 + 幂等去重)

```java
@Service
@RequiredArgsConstructor
public class IngestionService {
    private final PdfService pdf;
    private final AiClient ai;
    private final VectorService vector;
    private final StorageService storage;
    private final QuotaService quota;
    private final QuestionMapper questionMapper;
    private final MaterialGroupMapper materialMapper;
    private final IngestionJobMapper jobMapper;

    @Async("ingestExecutor")
    public void process(Long jobId) {
        IngestionJob job = jobMapper.selectById(jobId);
        if (job.getStatus() == 3) return;             // 已完成,幂等直接退出
        Long userId = job.getUserId();
        String scope = "user:" + userId;

        try {
            updateStatus(job, 1, 0);                   // 解析中

            // 1) 取 PDF,提文字 + 切出三类题
            byte[] pdfBytes = storage.get(job.getFileObjectKey());
            PdfParseResult parsed = pdf.parse(pdfBytes); // {普通题文本, 资料分析材料组+小题, 图推页码}

            // 2) 切题:LLM 自适应排版(分批+重叠),图推按特征话术识别
            List<RawQuestion> normals = ai.splitQuestions(parsed.normalText());
            int graphicCount = (int) normals.stream().filter(RawQuestion::isGraphic).count();

            int total = normals.size() + parsed.materialGroups().stream()
                    .mapToInt(g -> g.subQuestions().size()).sum();
            job.setTotalCount(total);
            job.setGraphicCount(graphicCount);
            updateStatus(job, 2, 0);                    // 入库中

            // 3) 取库内已有指纹做去重
            Set<String> existing = questionMapper.selectFingerprints(scope);

            int done = 0, dup = 0;
            // ---- 普通文字题 ----
            for (RawQuestion q : normals) {
                String fp = fingerprint(q.content());
                if (existing.contains(fp)) { dup++; bump(job, ++done, total); continue; }
                existing.add(fp);

                if (q.isGraphic()) {
                    // 图推题:多模态识规律(贵,扣多模态额度),原图存 MinIO
                    if (!quota.tryConsumeMultimodal(userId, 1)) { bump(job, ++done, total); continue; }
                    GraphicResult g = ai.recognizeGraphic(q.pageImage());   // qwen-vl
                    String imgKey = storage.putImage(scope, q.pageImage());
                    saveGraphic(scope, job, q, g, imgKey, fp);
                } else {
                    saveTextQuestion(scope, job, q, fp);                    // DeepSeek 分类+考点
                }
                bump(job, ++done, total);
            }

            // ---- 资料分析题(材料+小题+图表绑定) ----
            for (MaterialGroup mg : parsed.materialGroups()) {
                Long materialId = saveMaterial(scope, job, mg);            // 存材料+渲染页图
                for (SubQuestion sub : mg.subQuestions()) {
                    String fp = fingerprint(mg.material() + sub.content());
                    if (existing.contains(fp)) { dup++; bump(job, ++done, total); continue; }
                    existing.add(fp);
                    saveDataQuestion(scope, job, mg, sub, materialId, fp); // 带材料背景提炼考点
                    bump(job, ++done, total);
                }
            }

            quota.consumeUpload(userId, done - dup);    // 按实际入库题数扣上传额度
            job.setDoneCount(done); job.setDupCount(dup);
            updateStatus(job, 3, 100);                   // 完成
            job.setFinishedAt(LocalDateTime.now()); jobMapper.updateById(job);

        } catch (Exception e) {
            throw new RuntimeException("入库失败 job=" + jobId, e);  // 交给消费者重试/标失败
        }
    }

    // 文字题:DeepSeek 分类+考点 → 算向量 → 落库
    private void saveTextQuestion(String scope, IngestionJob job, RawQuestion q, String fp) {
        Classification c = ai.classify(q.content());     // 一次LLM:题型/二级/考点/难度/考点摘要
        float[] vec = ai.embed(c.topicSummary());        // 通义 embedding
        Question entity = buildQuestion(scope, job, q, c, fp);
        questionMapper.insert(entity);
        String vid = vector.upsert(entity.getId(), vec,
                Map.of("scope", scope, "l2", c.categoryL2())); // 向量带 metadata 供过滤
        entity.setVectorId(vid);
        questionMapper.updateById(entity);
    }

    private void bump(IngestionJob job, int done, int total) {
        int progress = total == 0 ? 100 : done * 100 / total;
        job.setDoneCount(done); job.setProgress(progress);
        jobMapper.updateById(job);                        // 进度落库,前端轮询可见
    }
    private void updateStatus(IngestionJob job, int status, int progress) {
        job.setStatus(status); job.setProgress(progress); jobMapper.updateById(job);
    }
    private String fingerprint(String text) {
        return DigestUtils.md5DigestAsHex(text.replaceAll("\\s","")
                .substring(0, Math.min(50, text.length())).getBytes());
    }
}
```

## 七、出题接口:纯 SQL 筛选,零 AI 成本

```java
@RestController
@RequestMapping("/api/practice")
@RequiredArgsConstructor
public class PracticeController {
    private final QuestionService questionService;

    /** "给我N道图形推理题" —— 走 idx_serve 索引,毫秒级 */
    @GetMapping("/questions")
    public R<List<QuestionVO>> getByType(
            @RequestParam(required = false) String l1,
            @RequestParam(required = false) String l2,
            @RequestParam(required = false) Integer difficulty,
            @RequestParam(defaultValue = "10") int limit,
            @RequestParam(defaultValue = "true") boolean excludeDone,
            @AuthUser Long userId) {
        return R.ok(questionService.serve(userId, l1, l2, difficulty, limit, excludeDone));
    }

    /** "再来几道和这题考点像的" —— 走向量检索 */
    @GetMapping("/similar/{questionId}")
    public R<List<QuestionVO>> similar(@PathVariable Long questionId,
                                       @RequestParam(defaultValue = "5") int k,
                                       @AuthUser Long userId) {
        return R.ok(questionService.findSimilar(userId, questionId, k));
    }

    /** 提交答案:判分 + 写做题记录 + 错题入错题本 */
    @PostMapping("/submit")
    public R<JudgeResult> submit(@RequestBody SubmitDTO dto, @AuthUser Long userId) {
        return R.ok(questionService.judgeAndRecord(userId, dto));
    }
}
```

## 八、可靠性要点(商业项目必做)

1. **幂等**:`process()` 入口判 `status==3` 直接退出;去重靠 `fingerprint`,重跑不会重复入库/重复扣费。
2. **进度可见**:每题 `bump()` 把 progress 落库,前端轮询 `GET /api/ingest/job/{id}` 即可。
3. **额度原子扣减**:`QuotaService` 用 Redis `DECRBY` 保证并发安全;多模态额度单独算(图推贵)。
4. **LLM 容错**:WebClient 配超时 + 有限重试 + 限流退避;JSON 解析做容错(去包裹、正则兜底)。
5. **scope 强校验**:出题/检索的 scope 永远后端拼,绝不信前端,私有题隔离是隐私红线。
6. **演进**:Redis Stream 撑不住了换 RabbitMQ;向量量大了把 Redis Stack 换 Qdrant——接口层不变。
