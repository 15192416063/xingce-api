# 行测相似题检索系统

## 这是什么
输入一道行测题，从你的题库里按"考点语义"找出最相似的题。
不是关键词匹配——比如"社区垃圾分类"和"乡镇河道治理"字面不同，但都考"基层治理"，本系统能关联起来。

## 三步走（按顺序）

### 准备
pip install pdfplumber langchain langchain-community langchain-openai chromadb
设置 API key：export OPENAI_API_KEY="你的key"  (Windows: $env:OPENAI_API_KEY="你的key")

### 第一步：切题（不花钱，先跑这个确认切对）
python step1_parse.py 你的试卷.pdf
→ 生成 questions.json。打开看看题切得对不对，再往下走。

### 第二步：建库（需要API）
python step2_build.py
→ 给每道题提炼考点、算向量，存进 xingce_db 文件夹。

### 第三步：检索（需要API）
python step3_search.py
→ 改最底下的 sample 换成你想查的题，运行即可。

## 核心设计（面试可讲）
1. 不是用整道题算向量，而是先用大模型提炼"考点摘要"再算向量，
   让"相似"精准落在考点维度，不被选项噪音干扰。
2. 用户输入走和建库完全相同的"提炼考点"流程，保证可比。
3. 设了相似度阈值，库里没有相似题时如实返回"没找到"，不硬凑。

## 已知局限（第一版有意做的减法）
- 图形推理题(题干全是图)自动过滤，不入库。
- 资料分析题(依赖共享材料)、申论题第一版跳过。
- 试卷无答案解析，故只能找相似题、不附答案。
